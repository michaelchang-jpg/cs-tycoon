# CS Tycoon 固定測試流程（回歸用）

## 目標
每次改版用同一套流程，快速判斷是否退化。

## 流程（約 2 分鐘）
1. 啟動 `main.tscn`
2. 觀察 30 秒：
   - 員工會在桌子與飲水機間切換
   - 行為文字與壓力值有更新
3. 再觀察 60 秒：
   - 壓力值會上升後回落
   - 公司資產持續成長
4. 開啟 Debug Overlay（預設開）：
   - State、Pos、Target、Stress Δ/s 都有值
5. 判定：
   - 只要有卡死 / 不移動 / 壓力不變動，記為 FAIL
